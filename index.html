sidebar_position: 1 title: "🚀 คู่มือ Vibe Coding: สร้างระบบบันทึกรายรับรายจ่าย (Serverless & Free Deploy)" sidebar_label: "Vibe Coding - รายรับรายจ่าย"​

ยินดีต้อนรับสู่หลักสูตรการเรียนรู้แบบ Vibe Coding ยุคใหม่! ในบทเรียนนี้เราจะพาคุณสร้าง "ระบบบันทึกรายรับรายจ่าย" ซึ่งเป็น Mini Project สำคัญเพื่อเตรียมพร้อมสู่การเป็นนักพัฒนาเว็บแอปพลิเคชันมืออาชีพ โดยใช้แนวคิด Serverless ที่สามารถ Deploy ขึ้น Vercel ได้ฟรี 100% ไม่มีค่าใช้จ่าย และจัดการผ่าน Git ได้อย่างเป็นระบบ

เราจะเน้นการใช้พลังของ AI วางแผนและเขียนโค้ดให้ ประหยัด Token สูงสุด และไม่มีค่าใช้จ่าย!

🧭 Roadmap การพัฒนา (The Vibe Coding Way)​

เราจะแบ่งการทำงานออกเป็น 4 ขั้นตอนหลัก เพื่อให้ได้ผลลัพธ์ที่ถูกต้องและไม่เสียค่าใช้จ่ายด้าน Token:

  1. Phase 1: Blueprint by Antigravity (Opus) - ให้ AI รุ่นใหญ่ช่วยวางแผนโครงสร้างโค้ดและการจัดเก็บข้อมูลแบบ Client-side (เน้นตรรกะ ไม่เน้นพิมพ์โค้ด)
  2. Phase 2: Code Generation by Cline (DeepSeek) - ย้ายมาทำงานบน VS Code ใช้โมเดลฟรี/ราคาประหยัดในการเขียนโค้ดทีละส่วน
  3. Phase 3: Version Control & Testing - จัดการซอร์สโค้ดด้วย Git และทดสอบระบบในเครื่อง
  4. Phase 4: Cloud Deployment - อัปโหลดระบบขึ้น GitHub และ Deploy สู่สาธารณะผ่าน Vercel ฟรี!

🎯 ขอบเขตของระบบ (Project Specification)​

ระบบบันทึกรายรับรายจ่ายของเราจะถูกออกแบบให้ตอบโจทย์การทำงานที่รวดเร็ว เบา และไม่มีความซับซ้อนของ Database หลังบ้าน:

1. ฟังก์ชันหลัก (Core Features)​

  • บันทึกรายการ: เพิ่มประเภท (รายรับ/รายจ่าย), หมวดหมู่, จำนวนเงิน, วันที่ และรายละเอียดเพิ่มเติม
  • แสดงผลและกรองข้อมูล: แสดงประวัติย้อนหลังในรูปแบบตารางหรือการ์ดที่อ่านง่าย พร้อมระบบกรองข้อมูล (เช่น ตามประเภท หรือ ช่วงวันที่)
  • จัดการข้อมูล (CRUD): สามารถแก้ไขและลบรายการได้ (มี JavaScript Popup ยืนยันก่อนลบ)
  • สรุปยอดรวม: คำนวณ ยอดคงเหลือ = รายรับรวม - รายจ่ายรวม แสดงให้เห็นชัดเจน

2. เทคโนโลยีที่เลือกใช้ (Tech Stack - Vercel Ready & No Database)​

  • Frontend/Framework: Next.js (App Router) หรือ React (Vite) ซึ่งรองรับการ Deploy บน Vercel ได้อย่างสมบูรณ์แบบ
  • Data Storage: LocalStorage (จัดเก็บข้อมูลภายในเบราว์เซอร์ของผู้ใช้โดยตรง ทำให้ระบบทำงานแบบไร้ฐานข้อมูลฝั่ง Server ประหยัดโค้ด และไม่มีค่าบริการ Database รายเดือน)

:::tip สิ่งที่ไม่ต้องทำ (Scope Limitation) โปรเจกต์นี้ไม่ต้องเชื่อมต่อฐานข้อมูล SQL/NoSQL ภายนอก และไม่ต้องทำระบบ Login ช่วยให้เราโฟกัสไปที่สถาปัตยกรรมของตัวแอปพลิเคชันและกระบวนการ Deploy ได้อย่างเต็มที่ :::

🧠 ขั้นตอนที่ 1: การใช้ Antigravity (Opus) วางพิมพ์เขียว​

เปิดใช้งาน Antigravity และเลือกโมเดล Opus จากนั้นให้คัดลอก Prompt ด้านล่างนี้ไปสั่งงาน เพื่อให้ได้โครงสร้างระบบที่แม่นยำที่สุดโดยไม่ทำลายโควตา Token ด้วยโค้ดขยะ

พรอมต์สำหรับออกแบบระบบ (System Design Prompt)​

ฉันกำลังทำ Mini Project "ระบบบันทึกรายรับรายจ่าย" ที่มีขอบเขตดังนี้:
- Tech Stack: Next.js (App Router) หรือ React (Vite) [เลือกตามเหมาะสม] ที่สามารถ Deploy บน Vercel ได้
- Database: ไม่มีฐานข้อมูลฝั่ง Server ให้ใช้ LocalStorage ในการเก็บข้อมูลแบบ CRUD
- ฟังก์ชัน: เพิ่ม/แสดง/แก้ไข/ลบ รายรับรายจ่าย, แสดงยอดคงเหลือ, และกรองข้อมูลได้

ช่วยทำหน้าที่เป็น Principal System Architect ออกแบบโครงสร้างโฟลเดอร์โปรเจกต์, 
โครงสร้างข้อมูล JSON Object ที่เหมาะสมสำหรับเก็บใน LocalStorage และลำดับฟังก์ชัน Component สำคัญที่ต้องสร้าง 
(ขอเฉพาะ Blueprint และคำอธิบายเชิงสถาปัตยกรรมเท่านั้น *ยังไม่ต้องเขียนโค้ดจริง*)

การจัดการผลลัพธ์​

เมื่อ Opus ส่งผลลัพธ์มาให้ ให้คุณสร้างไฟล์ชื่อ SYSTEM_SPEC.md ไว้ที่ Root ของโฟลเดอร์โปรเจกต์ของคุณ แล้วนำคำตอบที่ได้ไปบันทึกไว้ ไฟล์นี้จะเป็น "พิมพ์เขียว" ให้ Cline ทำงานต่อ

💻 ขั้นตอนที่ 2: ตั้งค่า Cline + DeepSeek บน VS Code (ฟรี 100%)​

ติดตั้ง Extension ชื่อ Cline บน VS Code จากนั้นเลือกเชื่อมต่อโมเดลฟรีเพื่อประหยัดค่าใช้จ่าย:

  1. ทางเลือกที่ 1 (แนะนำ): สมัคร OpenRouter แล้วเลือกใช้โมเดล deepseek-v3:free หรือ deepseek-r1:free
  2. ทางเลือกที่ 2 (Offline): รัน Ollama ในเครื่องคอมพิวเตอร์ของคุณ แล้วโหลดโมเดล deepseek-coder มาใช้งาน

เทคนิคคุมกำเนิด Token​

สร้างไฟล์ .clineignore ไว้ที่ Root โฟลเดอร์ แล้วใส่ค่าต่อไปนี้ เพื่อไม่ให้ Cline แอบอ่านโฟลเดอร์ขยะจน Token เต็ม:

node_modules/
.git/
.next/
dist/

🛠️ ขั้นตอนที่ 3: เริ่มส่งงานให้ Cline สร้างระบบ (Vibe Coding Time)​

ในหน้าต่างแชทของ Cline เราจะสั่งงานทีละขั้นตอน (Step-by-Step) และกด Clear Chat ทุกครั้งที่งานส่วนนั้นเสร็จสิ้นและทดสอบผ่าน เพื่อล้างประวัติ Context ให้ประหยัด Token สูงสุด

Step 3.1: สร้างโครงสร้างโปรเจกต์และ Component การจัดการข้อมูล (State & LocalStorage)​

สั่ง Cline ด้วยพรอมต์นี้:

อ้างอิงจากโครงสร้างใน `SYSTEM_SPEC.md` ช่วยสร้างแอปพลิเคชันในโปรเจกต์นี้ 
โดยเริ่มจากการเขียน Logic ในการอ่าน-เขียนข้อมูลรายรับรายจ่ายผ่าน LocalStorage 
พร้อมฟังก์ชันจัดการข้อมูลพื้นฐาน (เพิ่ม, แก้ไข, ลบ) ให้เรียบร้อยก่อน

(เมื่อ Cline พิมพ์เสร็จ ลองทดสอบรันในเครื่องผ่านคำสั่งรันเซอร์เวอร์ เช่น npm run dev เมื่อแน่ใจว่าทำงานได้ดี ให้กด Accept โค้ดใน Cline แล้วเปิดหน้าแชทใหม่)

Step 3.2: สร้าง UI และส่วนแสดงผล (Table, Summary & Filter)​

สั่ง Cline ในแชทใหม่ด้วยพรอมต์นี้:

ช่วยสร้างส่วนติดต่อผู้ใช้ (UI Components) ให้หน้าเว็บมีฟังก์ชันบันทึกข้อมูล, 
แสดงประวัติรายรับรายจ่ายในรูปแบบที่อ่านง่าย, มีปุ่มแก้ไข/ลบ (พร้อมการแจ้งเตือนยืนยันก่อนลบ), 
แสดงผลรวมยอดคงเหลือที่คำนวณแบบ Real-time และใส่ตัวกรองข้อมูล (Filter) ให้เรียบร้อย

📦 ขั้นตอนที่ 4: การนำโค้ดขึ้น Git และ GitHub​

หลังจากได้โค้ดที่รันในเครื่องคอมพิวเตอร์ผ่านแล้ว เราจะนำโปรเจกต์เข้าสู่ระบบ Version Control ของ Git เพื่อเตรียมนำขึ้น GitHub สำหรับเชื่อมต่อไปยัง Vercel ต่อไป

  1. Initialize Git และสร้าง .gitignore: ในโฟลเดอร์โปรเจกต์ของคุณ ตรวจสอบให้แน่ใจว่ามีไฟล์ .gitignore เพื่อป้องกันไม่ให้โค้ดขยะถูกอัปโหลดขึ้นไป (ปกติ Next.js/Vite จะสร้างมาให้แล้ว) จากนั้นเปิด Terminal แล้วพิมพ์:

    git init
    git add .
    git commit -m "feat: initial commit financial tracker application"

  2. นำขึ้น GitHub:

    • ไปที่เว็บไซต์ GitHub แล้วสร้าง Repository ใหม่ (ตั้งชื่อเป็นสาธารณะหรือส่วนตัวก็ได้)
    • คัดลอกคำสั่งในหน้า GitHub มาวางใน Terminal ของคุณ (ตัวอย่างเช่น):
    git remote add origin https://github.com/ชื่อผู้ใช้ของคุณ/ชื่อเรโปของคุณ.git
    git branch -M main
    git push -u origin main

🚀 ขั้นตอนที่ 5: การ Deploy ขึ้น Vercel (ฟรีและอัปเดตอัตโนมัติ)​

Vercel รองรับเฟรมเวิร์กอย่าง Next.js หรือ Vite เป็นอย่างดี และเนื่องจากเราไม่มีฐานข้อมูลฝั่งเซิร์ฟเวอร์ การทำงานทั้งหมดจะเกิดขึ้นที่หน้าบ้าน (Client-side) ทำให้การ Deploy ง่ายและเสถียรมาก

  1. สมัครและเข้าใช้งาน Vercel:
    • เข้าไปที่ Vercel จากนั้นล็อกอินด้วยบัญชี GitHub ของคุณ
  2. Import โปรเจกต์:
    • คลิกที่ปุ่ม "Add New..." จากนั้นเลือก "Project"
    • ระบบจะแสดงรายชื่อ Repository จาก GitHub ของคุณ ให้เลือกโปรเจกต์รายรับรายจ่ายที่เพิ่งสร้างขึ้นมาแล้วคลิก "Import"
  3. ตั้งค่าและสั่ง Deploy:
    • ระบบของ Vercel จะตรวจจับเฟรมเวิร์ก (Next.js หรือ Vite) ให้คุณโดยอัตโนมัติ
    • ในส่วนของ Environment Variables เนื่องจากแอปพลิเคชันของเราใช้ LocalStorage จึงไม่ต้องตั้งค่าฐานข้อมูลใดๆ
    • คลิกที่ปุ่ม "Deploy"
    • รอระบบทำงานประมาณ 1-2 นาที คุณจะได้ลิงก์เว็บไซต์สาธารณะ (เช่น your-project.vercel.app) ที่เปิดใช้งานได้ทันทีทั่วโลก!

:::success จุดเด่นของการเชื่อมต่อ GitHub + Vercel หลังจากนี้ หากคุณต้องการแก้ไขโค้ดหรือเพิ่มฟังก์ชันใหม่ คุณเพียงแค่สั่งงานผ่าน Cline ใน VS Code เมื่อแก้เสร็จแล้วสั่ง git commit และ git push ขึ้น GitHub โครงระบบของ Vercel จะสั่งตรวจสอบโค้ดและอัปเดตหน้าเว็บจริงให้คุณแบบอัตโนมัติทันทีโดยไม่ต้องเข้าเว็บ Vercel ซ้ำอีกรอบ! :::
